lfs-cache.yml¶

Source: https://github.com/tardis-sn/tardis/blob/master/.github/workflows/lfs-cache.yml

Reusable workflow responsible for caching the regression data. For the action that retrieves regression data see setup-lfs.

Works with both atom data and the complete regression data suite.

Inputs¶

This workflow can be called both via workflow_call and workflow_dispatch. See the source file for the full list of inputs.

Note

The allow-lfs-pull input controls whether the workflow performs a git lfs pull and caches the regression data. When allow-lfs-pull is false the workflow only checks whether the cache is present and up to date — this mode is used by tests to fail fast before tests actually run.

What it does¶

  1. Checkout the regression data repo without LFS.

  2. Build an LFS file list, used to derive the cache key.

  3. Validate the cache key with actions/cache/restore in lookup-only mode.

  4. Branch on allow-lfs-pull:

    • false — fail if the cache is missing.

    • true — run git lfs pull and save the LFS cache.
