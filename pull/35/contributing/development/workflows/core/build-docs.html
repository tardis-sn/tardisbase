build-docs.yml¶

Source: https://github.com/tardis-sn/tardis/blob/master/.github/workflows/build-docs.yml

Generates TARDIS documentation on master and pull requests, and deploys it to GitHub Pages via the gh-pages branch.

Jobs¶

tests-cache — uses lfs-cache to verify the regression data cache exists.

check-for-changes — the build-docs job only runs on pull requests if:

  1. there are changes inside /docs, or

  2. the build-docs label is applied.

It always runs on master and on workflow_dispatch.

build-docs

  1. Checkout code (PR or master).

  2. setup-env.

  3. setup-lfs — atom data only.

  4. Install tardisbase and pip packages.

  5. Install tardis in editable mode.

  6. Build the documentation with Sphinx.

  7. Set destination directory for gh-pages: pushes to the /pull folder for pull requests, or updates the root for master.

  8. Clean branch on workflow_dispatch — this is a nuclear option that wipes the entire gh-pages branch and starts fresh. See clean-docs.

  9. Deploy to GitHub Pages.

  10. Post a comment with success/failure of the job.
