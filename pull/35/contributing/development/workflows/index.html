TARDIS GitHub Actions Workflows¶

This section documents the GitHub Actions actions, reusable workflows, and core workflows maintained in the TARDIS repository under .github/workflows/. It also references related actions that live in the tardis-actions repository.

Actions¶

Small, reusable building blocks used across the core workflows.

  • setup-env.yml (source) — creates the conda environment from the lockfile and caches it for faster subsequent runs.

  • setup-lfs.yml (source) — restores the cached regression data / atom data LFS objects for use in a job.

Reusable Workflows¶

Workflows that are not triggered directly but are called by other workflows.

  • lfs-cache.yml (source) — checks whether regression data LFS objects are cached, and pulls + saves them if not.

Core Workflows¶

End-to-end workflows triggered by pushes, pull requests, schedules, or manual dispatch.

Testing¶

  • tests.yml (source) — runs the TARDIS test suite on Linux and macOS for every push / PR to master.

  • full-tests.yml (source) — runs the full test suite on a self-hosted runner (PR label-gated).

  • compare-regdata.yml (source) — regenerates regression data on a PR and posts a comparison report.

  • stardis-tests.yml (source) — pulls STARDIS and runs its tests against the current TARDIS commit.

Documentation¶

  • build-docs.yml (source) — builds the Sphinx docs and deploys them to GitHub Pages.

  • clean-docs.yml (source) — removes preview doc folders when PRs/branches are closed or deleted.

  • docstr-cov.yml (source) — tracks docstring coverage and updates a shields.io badge.

Release¶

  • pre-release.yml (source) — weekly cron that prepares a release PR with updated Zenodo metadata.

  • release.yml (source) — creates the GitHub Release, attaches lockfiles, and injects the DOI and changelog.

  • post-release.yml (source) — updates CHANGELOG.md, CITATION.cff, and credits after a release.

Benchmarks¶

  • benchmarks.yml (source) — runs airspeed velocity benchmarks and publishes results to tardis-benchmarks.

Code Quality¶

  • codestyle.yml (source) — runs ruff and posts a summary comment on PRs.

  • codespell.yml (source) — runs codespell on the docs/ tree.

  • mailmap.yml (source) — ensures PR authors have an entry in .mailmap.

Utility¶

  • util.yml (source) — PR housekeeping: LFS label warnings, GSoC label, first-time contributor welcome, ORCID check.

  • tardis-research-papers.yml (source) — monthly cron that refreshes the list of papers using TARDIS.
